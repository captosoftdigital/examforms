## SCRAPING STRATEGY BY SOURCE TYPE

### Type 1: Static HTML (60%)
**Examples**: UPSC, SSC, IBPS
**Method**: Scrapy with CSS/XPath selectors
**Frequency**: Daily

### Type 2: JavaScript-Heavy (25%)
**Examples**: SBI, Railway RRCs, some banks
**Method**: Puppeteer/Playwright (headless browser)
**Frequency**: Daily

### Type 3: PDF-Only Updates (10%)
**Examples**: Some state PSCs, universities
**Method**: PDF download + text extraction (PyPDF2, pdfplumber)
**Frequency**: Daily check for new PDFs

### Type 4: Portal-Based (5%)
**Examples**: NSP, some state portals
**Method**: API calls (if available) or form submission
**Frequency**: Weekly

## DATA QUALITY CHECKS

### Validation Rules
1. **Date Format**: Standardize all dates to YYYY-MM-DD
2. **Link Validation**: Check all official links are live
3. **Duplicate Detection**: Prevent same notification twice
4. **Completeness**: Flag if critical fields missing
5. **Freshness**: Alert if source not updated in 7 days

### Manual Review Queue
- Ambiguous dates (e.g., "first week of June")
- Broken official links
- Conflicting information from multiple sources
- New exam types not in database
